# Comparison Framework

## Purpose

DefenderBot comparisons are designed to give readers a clear, structured, and honest side-by-side view of crypto trading bots rather than a ranking based on affiliate payouts or marketing claims.

***

## Standard Comparison Criteria

Every comparison may include some or all of the following criteria depending on the products being compared:

| Criteria | What It Means |
|---|---|
| **Product purpose** | What the bot is designed to do and who it is built for |
| **Best use case** | Which type of user or strategy it suits best |
| **Supported exchanges** | Which exchanges it connects to and how well |
| **Bot types and strategies** | DCA, grid, HODL, arbitrage, signal-based, etc. |
| **Automation features** | Depth of automation, customization, backtesting |
| **Ease of use** | Onboarding clarity, UI quality, learning curve |
| **Pricing structure** | Plans, tiers, free trial availability, limitations |
| **Transparency** | How clearly the platform explains its features and risks |
| **Security disclosures** | API key handling, fund custody, security practices |
| **Educational support** | Guides, tutorials, community, documentation quality |
| **User level** | Beginner, intermediate, advanced, or professional |
| **Strengths** | Clear advantages based on the criteria above |
| **Weaknesses** | Limitations, gaps, or areas to be aware of |

***

## Comparison Principle

The goal is not to declare a winner unconditionally.

The goal is to help a specific type of user understand which product is better for their situation, strategy, and market conditions.

***

## Fairness Standard

- Claims must be traceable to official sources where possible
- If something cannot be verified, the comparison should say so
- Marketing language from the brand should not be treated as neutral fact
- Comparisons should reflect the product as it exists today, not outdated information
